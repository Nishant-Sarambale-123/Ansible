## ğŸ”· 2. Loops in Ansible

---

### ğŸ”¹ What are Loops?

Loops are used when you want to **repeat a task for multiple items** (like installing multiple packages, creating users, etc.).

ğŸ“˜ [Official Docs â€“ Loops](https://docs.ansible.com/ansible/latest/user_guide/playbooks_loops.html)

---

### ğŸ”¹ Basic Loop (with `loop`)

```yaml
- name: Install multiple packages
  apt:
    name: "{{ item }}"
    state: present
  loop:
    - nginx
    - curl
    - git
```

---

### ğŸ”¹ Loop with Dictionaries

```yaml

  user:
    name: "{{ item.name }}"
    shell: "{{ item.shell }}"
  loop:
    - { name: 'alice', shell: '/bin/bash' }

```

---

### ğŸ”¹ Loop with `with_items` (older syntax)

```yaml

  user:
    name: "{{ item }}"
  with_items:

    - user2
```

## ğŸ”· 3. Handlers in Ansible

---

### ğŸ”¹ What are Handlers?

**Handlers** are special tasks triggered by a `notify` statement **only when the preceding task makes a change** (idempotent behavior).

ğŸ“˜ [Official Docs â€“ Handlers](https://docs.ansible.com/ansible/latest/user_guide/playbooks_handlers.html)

---

### ğŸ”¹ Handler Example

```yaml
tasks:
  - name: Update nginx config
    copy:
      src: nginx.conf
      dest: /etc/nginx/nginx.conf
    notify: restart nginx

handlers:
  - name: restart nginx
    service:
      name: nginx
      state: restarted
```

âœ… Handler runs **only if the copy task results in a change.**

---

### ğŸ”¹ Multiple Notifies to Same Handler

```yaml
notify:
  - restart nginx
  - reload firewall
```

---

### ğŸ”¹ Handler Best Practices

* Place handlers under a `handlers:` section at the same level as `tasks:`
* Use consistent names for handlers
* Use handlers for service restarts, reloads, etc.

## âœ… Interview Questions & Scenario-Based Q\&A

---

### â“ Q1. What is the difference between `when` and `register`?

**Answer:**

* `when` is used to conditionally run a task
* `register` stores the result of a task

---

### â“ Q2. Scenario: Only install a package if it's not already installed.

```yaml
- name: Check if nginx exists
  shell: which nginx
  register: nginx_exists
  ignore_errors: true

- name: Install nginx
  apt:
    name: nginx
    state: present
  when: nginx_exists.rc != 0
```

---

### â“ Q3. How do you install multiple packages using a loop?

```yaml
- name: Install required packages
  apt:
    name: "{{ item }}"
    state: present
  loop:
    - nginx
    - git
    - curl
```

---

### â“ Q4. What is the use of handlers?

**Answer:**
Handlers are triggered only when notified by another task, and they run **after all tasks are completed**. Commonly used for service restarts or reloading configs.

---

### â“ Q5. Scenario: You updated an nginx config. How to restart the service only if the config changed?

```yaml
- name: Update config
  template:
    src: nginx.conf.j2
    dest: /etc/nginx/nginx.conf
  notify: restart nginx

handlers:
  - name: restart nginx
    service:
      name: nginx
      state: restarted
```

---

### â“ Q6. Can you run a handler immediately?

**Answer:**
Yes, use `meta: flush_handlers`:

```yaml
- meta: flush_handlers
```
